NetBrain DS · Overview
Overview
NetBrain's design system builds on Kendo UI's Meridian theme token layer. Foundation defines the shared visual vocabulary across every component: color, typography, spacing, border radius, and elevation. We adopt Kendo's semantic token names and override only what's needed, keeping the system predictable and easy to maintain.
Design Tokens --kendo-color-*

Named values that represent every visual decision — color, spacing, radius, typography. Think of them as variables: change one value and every component using it updates automatically. Color tokens are semantic (e.g. "primary", "error") so the system knows the intent, not just the color.

Examples--kendo-color-primary · --kendo-spacing-4 · --kendo-border-radius-md
Component Tokens $kendo-{component}-*

SCSS variables scoped to a specific component — button, input, chip, etc. Each references a design token, creating the link between "what this component looks like" and "which global value drives it." Updating a design token automatically propagates to every component variable that references it.

Examples$kendo-button-bg · $kendo-button-hover-bg · $kendo-input-border
Color
Color System
Colors are organized into variable groups. The Main group (base) covers neutral UI surfaces, borders, and body text — it has no single base color but a set of fixed surface values. All other groups — primary, secondary, tertiary, success, warning, error, info — each have one editable BASE value; all interactive states (hover, active, subtle, on-surface) derive automatically from it via CSS oklch(from var(…)) relative-color syntax. Click any BASE chip to edit live.
Type
Typography
NetBrain uses Inter as the primary typeface and JetBrains Mono for code and token labels. The component font size base is 14px (0.875rem), not 16px.
Layout
Spacing
4px base unit. All values are multiples of --kendo-spacing-base (0.25rem). Changing the base shifts the entire scale.
Layout
Border Radius
All values derive from --kendo-border-radius-base = 0.375rem (6px).
Effects
Elevation
Box shadow layers defining depth hierarchy — from card surfaces to modal dialogs.
